# CIS-476-DriveShare-Platform`
    DriveShare is a peer-to-peer car rental platform will allow users to list, search, and book vehicles. 
    The system simply establishes object-oriented design and design patterns including Singleton, Observer, Mediator, Builder, Proxy, and Chain of Responsibil

## **Features:**

#### User System
    User sign-up and sign-in.
    Password reset using 3 security questions.
#### Car Management
    Updating car pricing and the availabilty.
    Owners can list the cars and their detais with ( model number and price).
#### Search and Booking
    Stopping for overlapping during bookings
    Renters can view and will able to book cars.
#### Notifications
    Renters will notified when price changes or any updates has been made.
    Renters can watch cars
#### Payment Systems
    Proxy pattern simply will be used to secure interaction
#### Messaging and Recovery
    Chain of Responsibility will act as password recovery.

## **6 Design Patterns Used:**

1. [ ] Singleton-------------------------------> for User session management
2. [ ] Observer-------------------------------> for car price notifications
3. [ ] Builder------------------------------> for car object creation
4. [ ] Proxy------------------------------> for payment processing
5. [ ] Chain of Responsibility----------> for password recovery
6. [ ] Mediator-----------------------> for UI interaction mange.
